# AI Core 实现验证文档（家计簿场景）

---

# 一、用户需求（自然语言版）

我想做一个智能家计簿系统。

我会通过自然语言输入消费信息，例如：

* 吃饭花了3000日元，两个人，在博多一兰拉面
* 今天买了咖啡500日元，还有买了书1200日元
* 上周末和家人去超市买东西一共花了8000日元

系统需要做到：

## 1. 自动理解并拆分消费

* 一次输入可能包含多条消费
* 每条消费需要拆分成独立记录

## 2. 自动结构化字段

每条记录需要包含：

* 时间（支持模糊时间：今天、上周、5月第一周）
* 地点（餐厅/城市/店名）
* 消费内容
* 金额
* 人数（可选）
* 消费人（我/家人/朋友）
* 消费类别（餐饮/购物/娱乐等）

## 3. 支持后续查询分析

支持自然语言查询：

* 5月份第一周一共花了多少钱？
* 这个月餐饮花了多少？
* 我个人花了多少？
* 家人花了多少？
* 咖啡一共花了多少？
* 博多地区消费总额是多少？

## 4. 支持多维度统计

系统应支持：

* 按时间（天/周/月）
* 按消费人
* 按消费类别
* 按地点
  进行聚合分析

# 二、Core 考察成果 / 验证项

## 1. 功能完成度

### 验证项

* 是否能正确理解自然语言输入
* 是否能拆分多条消费记录
* 是否能提取完整字段
* 是否支持模糊时间解析
* 是否支持自然语言查询
* 是否能进行多维度聚合统计

### 验收标准

* 多次输入结果稳定
* 查询结果与数据一致

---

## 2. 模型 / 技术 / 数据库选型

### 模型

* 输入解析模型
* 工作流规划模型
* 查询解析模型
* 代码 / Blueprint 生成模型

### 技术

* FastAPI（API入口）
* Workflow（LangGraph / 自定义DAG）
* Adapter模式（模型/工具）

### 数据库

* PostgreSQL（主存储）
* 可选向量数据库（记忆检索）

---

## 3. 性能与成本

### 验证项

* 单条输入处理时间
* 查询响应时间
* 工作流执行时间
* 模型调用成本
* 并发能力

## 4. 工作流与逻辑编排

### 验证项

* 是否支持失败重试
* 是否支持动态调整流程

## 8. 异常处理能力

* 查询冲突处理

# 三、验收总结标准（AI判断用）

AI在生成系统代码后，应满足：

1. 功能完整
2. 结构清晰（模块化）
3. Workflow 可执行
4. Blueprint 可复用
5. Agent 可生成
6. 数据可查询
7. 支持自然语言交互

# 四、用途说明

该文档用于：

* AI 自动生成代码的输入规范
* Core 架构验证标准
* 系统验收基准

AI 在生成代码时必须：

* 覆盖所有验证项
* 输出可运行系统
* 提供模块化结构
* 支持扩展
