# 2026-04-20 Homework

## 主题

把 NestHub 现在已经具备的本地模型、知识库、运行时经验和生成资产，沉淀成真正可持续积累、可迁移、可训练的“私有主脑”。

核心判断：

1. 先做成“自己的系统”，再做成“自己的模型”。
2. 长期知识默认放在外部存储，不直接烤进模型权重。
3. 模型是推理器，知识库、策略库、经验库、训练集才是长期资产。

## 一、私有主脑分层架构

### L1. 会话工作记忆

目标：承载当前任务正在发生的上下文，不做长期真相存储。

当前落点：

1. `session_store`
2. `session_state`
3. `ContextManager`
4. `received/<session_id>/`

应该存放：

1. 当前会话上传的文档 / 图片 / 附件
2. 待处理请求，如 `pending_document_request`
3. 当前任务步骤输出
4. 本轮临时推理上下文

不应该存放：

1. 长期家庭知识
2. 规则偏好
3. 可复用经验结论

### L2. 程序化长期记忆

目标：沉淀 NestHub 的规则、意图知识、关键词、策略偏好、执行习惯。

当前落点：

1. `nethub_runtime/core/config/semantic_policy.json`
2. `semantic_policy_memory.sqlite3`
3. `SemanticPolicyStore`
4. `runtime_intent_knowledge`

应该存放：

1. 文档/图片分析关键词和业务表达
2. 用户偏好输出格式
3. 领域规则、字段映射、完成条件
4. 可激活的运行时知识候选

这是主脑的“脑回路层”，比模型更像真正属于系统自己的部分。

### L3. 结构化事实知识

目标：沉淀家庭、联系人、设备、时间线、资料结论等长期事实。

当前落点：

1. `InformationAgentService`
2. 信息智能体生成结果
3. `vector_store`
4. 未来可补充的 profile / entity store

应该存放：

1. 家庭成员信息
2. 外部联系人信息
3. 设备与空间信息
4. 文档总结后的结构化事实
5. 图片分析后的结构化识别结果

### L4. 执行经验与自优化记忆

目标：积累“怎么做更容易成功”。

当前落点：

1. `RuntimeLearningStore`
2. `runtime_learning_store.record_attempt`
3. `runtime_learning_store.record_execution_outcome`

应该存放：

1. 某类任务失败的原因
2. 哪种模型/工具组合成功率高
3. 需要修复几轮才能成功
4. 哪些能力缺口会反复出现

这一层是私有主脑的“自我优化层”。

### L5. 模型权重层

目标：在前三层资产稳定以后，把高质量样本蒸馏成真正属于自己的本地模型能力。

当前状态：

1. 还不应该作为第一优先级
2. 应该等训练集和知识沉淀机制稳定后再进入 LoRA / SFT

结论：

真正私有化的顺序是：

1. 先资产化
2. 再数据化
3. 最后才是模型化

## 二、私有主脑的核心闭环

### 1. 任务执行

NestHub 接收用户请求，走现有的：

1. `IntentAnalyzer`
2. `TaskDecomposer`
3. `WorkflowPlanner`
4. `ExecutionCoordinator`
5. `ResultIntegrator`

### 2. 结果评估

基于现有：

1. `RuntimeOutcomeEvaluator`
2. `UserGoalEvaluator`
3. `RuntimeFailureClassifier`

把任务结果分为：

1. 成功
2. 部分成功
3. 失败
4. 修复后成功

### 3. 归因分类

把结果归因到这四类：

1. 规则缺失
2. 事实知识缺失
3. 工具/模型能力缺失
4. 输出风格/用户偏好偏差

### 4. 回写长期资产

按归因写回：

1. 规则缺失 -> `SemanticPolicyStore`
2. 事实知识 -> `InformationAgentService` / `vector_store`
3. 能力经验 -> `RuntimeLearningStore`
4. 偏好变化 -> 用户偏好记忆或 semantic policy overlay

### 5. 下次优先重用

下一次执行前，优先查：

1. runtime intent knowledge
2. runtime learning summary
3. information agent facts
4. vector retrieval context

目标：避免每次重新试错。

## 三、第一阶段必须补的能力

### A. 高质量结果自动入库

当前缺口：

虽然系统能记录执行结果，但还没有统一判断“这条结果值不值得长期保留”。

建议新增一个统一策略：`memory_promotion_policy`。

拟判定维度：

1. 是否为用户长期事实
2. 是否为可复用规则
3. 是否为高质量执行经验
4. 是否为可训练样本

建议新增模块：

1. `nethub_runtime/core/services/memory_promotion_service.py`

建议核心接口：

```python
promote_execution_result(
		*,
		task: TaskSchema,
		context: CoreContextSchema,
		execution_result: dict[str, Any],
) -> dict[str, Any]
```

职责：

1. 判定哪些结果应该进入长期知识
2. 把结论拆分到合适的存储层
3. 返回入库摘要，供 trace 和后续学习使用

### B. 结果蒸馏为训练样本

目标：不是所有对话都拿去训练，只导出高价值样本。

建议新增模块：

1. `nethub_runtime/core/services/training_dataset_export_service.py`

建议输出目录：

1. `generated/datasets/sft/`
2. `generated/datasets/preferences/`

建议样本来源：

1. 文档总结成功结果
2. 图片内容分析成功结果
3. 信息智能体 schema 规划成功结果
4. 修复前失败、修复后成功的配对样本

建议样本结构：

```json
{
	"input": {
		"user_text": "请分析这张图片里的动物是什么",
		"context": {...},
		"retrieved_knowledge": [...]
	},
	"output": {
		"answer": "图片里是一只猫。",
		"final_output": {...}
	},
	"quality": {
		"outcome": "success",
		"repair_iterations": 0,
		"user_goal_satisfied": true
	}
}
```

### C. 主脑知识资产总览

目标：让系统能回答“我已经学到了什么”。

建议新增一个汇总接口，聚合：

1. Semantic policy active overlays
2. Runtime learning summary
3. Information agent entity counts
4. Vector store item counts
5. 可导出训练样本数量

建议新增位置：

1. `AICore.inspect_private_brain_summary()`

## 四、实现顺序

### Phase 1

目标：先把“结果自动入库”跑通。

实施：

1. 增加 `memory_promotion_service.py`
2. 在 `core_engine.handle()` 完成后调用
3. 先支持文档总结、图片内容分析、信息采集三类任务

### Phase 2

目标：把运行时经验从记录升级为复用。

实施：

1. 扩展 `RuntimeLearningStore.lookup_solution()` 的接入点
2. 在任务规划或执行前优先查经验
3. 缩短重复失败路径

### Phase 3

目标：把高质量历史结果导出成训练样本。

实施：

1. 增加 `training_dataset_export_service.py`
2. 导出 SFT 和 preference 数据
3. 落到 generated artifacts

### Phase 4

目标：再决定是否进入本地模型微调。

实施前提：

1. 至少已经积累稳定的结构化高质量样本
2. 可以追踪样本来源与质量
3. 能区分知识、偏好、经验、输出格式四种数据用途

## 六、验收标准

完成第一阶段后，系统应该能做到：

1. 一次高质量文档总结结果可以被判定为长期知识资产
2. 一次高质量图片分析结果可以被拆成结构化事实或训练样本候选
3. 一次成功的能力修复结果可以进入 runtime learning
4. 主脑可以输出“我目前积累了哪些知识、偏好、经验”
5. 更换底层本地模型后，以上资产仍然保留并可继续复用
